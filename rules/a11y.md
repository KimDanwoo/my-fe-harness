# 웹 접근성 (a11y) — 핵심

처음부터 기본값으로. WCAG AA 지향, 아래만 지켜도 대부분 해결된다.

```tsx
// ❌ 클릭 가능한 div — 키보드·스크린리더 접근 불가
<div onClick={close}>✕</div>
// ✅ 네이티브 요소 + 접근 가능한 이름
<button type="button" onClick={close} aria-label="닫기">✕</button>
```

- 클릭·이동·입력은 네이티브 요소로(`button`/`a[href]`/`input`). 아이콘만 있으면 `aria-label`, 이미지엔 `alt`.
- 키보드만으로 조작 가능 + 가시적 포커스 유지(`outline:none` 단독 금지). 모달은 포커스 가두고 Esc·복귀 처리.
- 폼: `label`-`input` 연결, 에러는 `aria-invalid`+`aria-describedby`(비동기는 `role="alert"`).
- 제목 계층(h1→h2), `<html lang>`, 대비 4.5:1, 색만으로 정보 전달 금지.
- 네이티브 시맨틱을 `role`로 덮어쓰지 않는다 — 나쁜 ARIA보다 ARIA 없음이 낫다.
